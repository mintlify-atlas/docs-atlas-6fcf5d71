The `@loopback/authentication` package gives you a pluggable authentication layer for your LoopBack 4 application. You decorate controller methods with `@authenticate` to declare that a route requires a verified identity, then register one or more authentication strategies — such as the built-in JWT extension — that handle the actual credential verification. The authenticated user profile is then available via dependency injection throughout the rest of the request lifecycle.

## Install the packages

<Steps>
  <Step title="Install the core authentication package">
    ```bash
    npm install @loopback/authentication
    ```
  </Step>
  <Step title="Install the JWT extension (optional but recommended for most apps)">
    ```bash
    npm install @loopback/authentication-jwt
    ```

    <Warning>
      `@loopback/authentication-jwt` is an experimental package (versioned `0.x.y`). Its API may change between minor releases. For production use, review its source and consider replacing the default `UserService` with your own implementation.
    </Warning>
  </Step>
</Steps>

## Register the authentication component

Mount `AuthenticationComponent` in your `application.ts` before adding any strategies. If you are using the JWT extension, also mount `JWTAuthenticationComponent` and bind a datasource for user persistence.

```typescript
import {AuthenticationComponent} from '@loopback/authentication';
import {
  JWTAuthenticationComponent,
  UserServiceBindings,
} from '@loopback/authentication-jwt';
import {DbDataSource} from './datasources';

export class TodoListApplication extends BootMixin(
  ServiceMixin(RepositoryMixin(RestApplication)),
) {
  constructor(options: ApplicationConfig = {}) {
    super(options);

    // Mount the core authentication system
    this.component(AuthenticationComponent);
    // Mount the JWT strategy, token service, and user service
    this.component(JWTAuthenticationComponent);
    // Bind a datasource so the JWT component can persist users
    this.dataSource(DbDataSource, UserServiceBindings.DATASOURCE_NAME);
  }
}
```

<Note>
  If your application uses the default middleware-based sequence (generated by `lb4 app` in recent versions), you do not need to add an authentication action manually — the component wires it up automatically.
</Note>

## Protect routes with @authenticate

Place `@authenticate('jwt')` (or any registered strategy name) on a controller method to require a valid credential. After successful authentication, inject the current user profile with `SecurityBindings.USER`.

```typescript
import {authenticate, TokenService} from '@loopback/authentication';
import {
  Credentials,
  MyUserService,
  TokenServiceBindings,
  UserServiceBindings,
} from '@loopback/authentication-jwt';
import {inject} from '@loopback/core';
import {get, post, requestBody} from '@loopback/rest';
import {SecurityBindings, securityId, UserProfile} from '@loopback/security';

export class UserController {
  constructor(
    @inject(TokenServiceBindings.TOKEN_SERVICE)
    public jwtService: TokenService,
    @inject(UserServiceBindings.USER_SERVICE)
    public userService: MyUserService,
    @inject(SecurityBindings.USER, {optional: true})
    public user: UserProfile,
  ) {}

  // Public route — no decorator needed
  @post('/users/login')
  async login(
    @requestBody(CredentialsRequestBody) credentials: Credentials,
  ): Promise<{token: string}> {
    const user = await this.userService.verifyCredentials(credentials);
    const userProfile = this.userService.convertToUserProfile(user);
    const token = await this.jwtService.generateToken(userProfile);
    return {token};
  }

  // Protected route — requires a valid JWT in the Authorization header
  @authenticate('jwt')
  @get('/whoAmI')
  async whoAmI(
    @inject(SecurityBindings.USER)
    currentUserProfile: UserProfile,
  ): Promise<string> {
    return currentUserProfile[securityId];
  }
}
```

<Tip>
  When only some methods in a controller are decorated with `@authenticate`, declare `SecurityBindings.USER` as optional in the constructor (`{optional: true}`) to avoid a binding error on unauthenticated routes. Alternatively, inject it directly in the method signature instead.
</Tip>

### Decorator syntax reference

`@authenticate` accepts a strategy name as a string, an options object, or multiple strategies:

```typescript
// Single strategy by name
@authenticate('jwt')

// Single strategy with options
@authenticate({strategy: 'basic', options: {session: false}})

// Multiple strategies tried in order
@authenticate('jwt', 'basic')

// Multiple strategies, each with options
@authenticate(
  {strategy: 'jwt', options: {algorithms: ['HS256']}},
  {strategy: 'apikey', options: {header: 'x-api-key'}},
)
```

## Issue tokens: login endpoint

The `UserService` and `TokenService` interfaces from `@loopback/authentication` define the contract for verifying credentials and producing tokens.

```typescript
// From @loopback/authentication
export interface UserService<U, C> {
  verifyCredentials(credentials: C): Promise<U>;
  convertToUserProfile(user: U): UserProfile;
}

export interface TokenService {
  verifyToken(token: string): Promise<UserProfile>;
  generateToken(userProfile: UserProfile): Promise<string>;
  revokeToken?(token: string): Promise<boolean>;
}
```

`JWTAuthenticationComponent` provides default implementations of both services. Inject them by binding key:

| Binding key | What it provides |
|---|---|
| `TokenServiceBindings.TOKEN_SERVICE` | `TokenService` — generates and verifies JWTs |
| `UserServiceBindings.USER_SERVICE` | `UserService` — looks up users and validates passwords |

## Configure token secrets and expiry

Override the defaults in `application.ts` to set your own secret and expiration:

```typescript
import {
  TokenServiceBindings,
  RefreshTokenServiceBindings,
} from '@loopback/authentication-jwt';

// In your application constructor:
this.bind(TokenServiceBindings.TOKEN_SECRET).to('my-jwt-secret');
this.bind(TokenServiceBindings.TOKEN_EXPIRES_IN).to('3600'); // seconds

// For refresh tokens:
this.bind(RefreshTokenServiceBindings.TOKEN_SECRET).to('my-refresh-secret');
this.bind(RefreshTokenServiceBindings.TOKEN_EXPIRES_IN).to('86400');
```

## Add a custom sequence (non-middleware apps only)

If your application uses a custom `SequenceHandler` rather than `MiddlewareSequence`, you must inject and call the authentication action yourself:

```typescript
import {
  AuthenticateFn,
  AuthenticationBindings,
  AUTHENTICATION_STRATEGY_NOT_FOUND,
  USER_PROFILE_NOT_FOUND,
} from '@loopback/authentication';

export class MySequence implements SequenceHandler {
  constructor(
    @inject(AuthenticationBindings.AUTH_ACTION)
    protected authenticateRequest: AuthenticateFn,
    // ...other injected actions
  ) {}

  async handle(context: RequestContext) {
    try {
      const {request, response} = context;
      const route = this.findRoute(request);
      // Run authentication before invoking the controller
      await this.authenticateRequest(request);
      const args = await this.parseParams(request, route);
      const result = await this.invoke(route, args);
      this.send(response, result);
    } catch (error) {
      if (
        error.code === AUTHENTICATION_STRATEGY_NOT_FOUND ||
        error.code === USER_PROFILE_NOT_FOUND
      ) {
        Object.assign(error, {statusCode: 401});
      }
      this.reject(context, error);
    }
  }
}
```

Then bind the sequence in `application.ts`:

```typescript
this.sequence(MySequence);
```

## Write a custom authentication strategy

Implement the `AuthenticationStrategy` interface when you need a strategy beyond JWT — for example, API key, OAuth2, or session-based auth.

```typescript
import {AuthenticationStrategy} from '@loopback/authentication';
import {Request, RedirectRoute} from '@loopback/rest';
import {UserProfile} from '@loopback/security';

export class ApiKeyStrategy implements AuthenticationStrategy {
  name = 'apikey';

  async authenticate(
    request: Request,
  ): Promise<UserProfile | RedirectRoute | undefined> {
    const apiKey = request.headers['x-api-key'];
    if (!apiKey) {
      throw new HttpErrors.Unauthorized('API key is missing');
    }
    // Validate the key and return a UserProfile...
  }
}
```

Register the strategy in `application.ts` using `registerAuthenticationStrategy`:

```typescript
import {registerAuthenticationStrategy} from '@loopback/authentication';

export class MyApplication extends BootMixin(
  ServiceMixin(RepositoryMixin(RestApplication)),
) {
  constructor(options?: ApplicationConfig) {
    super(options);
    this.component(AuthenticationComponent);
    registerAuthenticationStrategy(this, ApiKeyStrategy);
  }
}
```

`AuthenticationStrategy` requires two members:

| Member | Type | Description |
|---|---|---|
| `name` | `string` | Unique identifier referenced in `@authenticate('name')` |
| `authenticate` | `(request: Request) => Promise<UserProfile \| RedirectRoute \| undefined>` | Verifies the request and returns a user profile, a redirect, or `undefined` |

If the credentials are invalid, throw an error from `authenticate()`. If the credentials are simply absent and your strategy should be skipped, return `undefined`.

## Customize the user model

The default `MyUserService` bundled with `@loopback/authentication-jwt` is a reference implementation. Replace it with your own to control the user model and credential logic:

```typescript
import {UserService} from '@loopback/authentication';
import {securityId, UserProfile} from '@loopback/security';
import {MyUser} from '../models';
import {MyUserRepository} from '../repositories';

export type Credentials = {email: string; password: string};

export class CustomUserService implements UserService<MyUser, Credentials> {
  constructor(
    @repository(MyUserRepository) public userRepository: MyUserRepository,
  ) {}

  async verifyCredentials(credentials: Credentials): Promise<MyUser> {
    const user = await this.userRepository.findOne({
      where: {email: credentials.email},
    });
    if (!user) throw new HttpErrors.Unauthorized('Invalid email or password.');
    // verify password...
    return user;
  }

  convertToUserProfile(user: MyUser): UserProfile {
    return {
      [securityId]: user.id.toString(),
      name: user.username,
      id: user.id,
      email: user.email,
    };
  }
}
```

Then bind the custom service in `application.ts`:

```typescript
import {UserServiceBindings} from '@loopback/authentication-jwt';
this.bind(UserServiceBindings.USER_SERVICE).toClass(CustomUserService);
```
